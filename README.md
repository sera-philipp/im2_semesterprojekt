Projekt-Doku: ufä

Projektidee: Ich habe eine interaktive Website namens "ufä" entwickelt. Die Website soll Gleitschirmfliegenden dabei helfen, passende Fluggebiete auszuwählen und wichtige Informationen zu den jeweiligen Orten schnell zu erfassen. Im Zentrum steht eine Schweizerkarte, auf der verschiedene Regionen mit animierten Lottie-Buttons markiert sind. Über diese Buttons können die einzelnen Fluggebiete geöffnet werden. 

Im HTML habe ich die Grundstruktur der Seite aufgebaut: Navigation, Hero-Bereich mit Karte, einzelne Fluggebiet-Sections und einen Detailbereich für zusätzliche Informationen. Im CSS habe ich das visuelle Erscheinungsbild gestaltet. Dazu gehören das wiederholende Grid-Hintergrundbild, die farbliche Unterscheidung der Regionen sowie das Layout der Spot-Links. Die einzelnen Regionen haben alle eigene Farben erhalten. 

Die Lottie-Buttons wurden absolut auf der Karte positioniert, damit sie auch bei skalierender Karte an der richtigen Stelle bleiben. Das stellte auch eine grosse Herausfordeurng dar. Denn die Karte musste in einen relativen Wrapper gelegt werden, während die Buttons absolut mit Prozentwerten positioniert wurden. Hier hat mit der Einsatz von Codex sehr geholfen und ich konnte es am Schluss noch manuell optimieren.

Mit JavaScript habe ich die Interaktionen der Website umgesetzt. Beim Klick auf einen Lottie-Button wird nur das passende Fluggebiet eingeblendet. Klickt man danach auf eine Ortschaft, erscheint ein Detailbereich mit Windwerten, Informationen zur Gondelbahn und einem Webcam-Link. Für die aktuellen Windwerte verwende ich die Open-Meteo API. Die Daten werden geladen und anhand eines Indexes der jeweiligen Ortschaft zugeordnet. Zusätzlich wird ein Pfeil angezeigt, der sich entsprechend der Windrichtung dreht. Zwischendurch bestand ein Ladefehler, dieser schien aber extern bestanden zu haben (nach ein paar Stunden funktionierte es wieder).
Da es für Gondelbahn-Status und Webcam-Informationen keine passende offene API gab, habe ich eine eigene lokale JSON-Datei erstellt. Diese enthält pro Ortschaft Informationen zu Öffnungszeiten, Revisionen, Bemerkungen und Webcam-Links. Das JavaScript berechnet daraus, ob eine Gondelbahn aktuell als geöffnet oder geschlossen angezeigt wird. Auch hier habe ich viel mit Codex gearbeitet, da mir das Know-How dazu noch fehlt. Ich freue mich auf das kommende 3. Semester, da kann ich dann die bestehende Datenbank ausbauen und optimieren.

Ein kleiner, aber typischer Stolperstein war ausserdem der Browser-Cache. Nachdem ich mehrmals meine Anpassungen gepusht habe, aber die Änderungen nicht sichtbar waren, fiel mir dieser Fehler dann wieder erneut auf...

Fazit: Besonders wichtig war mir für dieses Projekt, dass die Website nicht nur schön aussieht, sondern auch funktional ist. Durch die Kombination aus Karte, animierten Buttons, API-Daten und eigener JSON-Datenstruktur ist eine tolle interaktive Webseite entstanden. Ich bin zufrieden mit dem Ergebnis und werde versuchen, meinen Javascript-Code zukünftig etwas selbständiger zu schreiben und weniger von KI abhängig zu machen. Ich merkte hier, dass mir der Transfer von Theorie (die ich durchaus gut verstanden habe) in die Praxis wirklich schwer viel. Auch vertiefen muss ich nochmals mein Verständnis von GitHub und "commit" / "push".
